# 小程序关键流程回归清单

适用范围：`mp-weixin-jiajiao`  
目标：每次发版前，快速验证核心链路是否可用，避免登录、发布需求、接单、订单流转回归。

## 1. 环境准备

- 后端服务已启动，接口可访问。
- 小程序 `baseUrl` 指向当前测试环境。
- 测试账号准备：
  - 家长账号 1 个
  - 教员账号 1 个（含资料完整 + 审核通过）
  - 教员账号 1 个（资料不完整或审核未通过）

## 2. 鉴权与登录

- [ ] 游客进入首页与需求页，页面可正常渲染，无崩溃。
- [ ] 游客进入订单页，显示游客态与登录引导。
- [ ] 家长登录成功后，返回目标页（带 redirect 时生效）。
- [ ] 教员登录成功后，身份相关页面显示正确（可接单视角）。
- [ ] token 失效后，请求返回 401 时，出现重新登录引导，不出现死循环跳转。

## 3. 发布需求（家长）

- [ ] 必填项校验生效：标题/描述/年级/地址/薪资。
- [ ] 标题超长与薪资异常范围会被拦截并提示。
- [ ] 学科、区域未选择时不可提交。
- [ ] 上传图片中按钮禁用生效，避免重复提交。
- [ ] 发布成功后跳转到“需求/订单相关页”且能看到新记录。
- [ ] 定位失败时有可执行引导（去设置/地图手动选点），不是仅报错。

## 4. 需求广场与接单（教员）

- [ ] 需求列表可加载，筛选（学科/区域/预算/高级筛选）可用。
- [ ] 教员资料审核通过时，“我要接单”按钮可用且可成功接单。
- [ ] 教员资料未通过/未完善时，接单被拦截并有明确引导。
- [ ] 同一需求被他人抢先接单时，提示冲突信息（不出现静默失败）。

## 5. 订单流转

- [ ] 家长侧可见“需求池 + 订单池”合并列表，阶段文案为中文且正确。
- [ ] 教员可执行：待确认 -> 进行中 -> 已完成。
- [ ] 家长可执行：待确认/进行中 -> 取消。
- [ ] 非法状态操作按钮应禁用或被拦截提示。
- [ ] 订单轨迹页可正常打开并显示时间线。

## 6. 页面状态与体验

- [ ] loading 态：首次加载有骨架/加载提示。
- [ ] empty 态：无数据时有明确空态文案。
- [ ] error 态：请求失败时可重试且文案可理解。
- [ ] 关键按钮状态完整：默认/禁用/loading。

## 7. 发布前结论

- [ ] 阻断问题（崩溃、主流程不可用）= 0
- [ ] 主要问题（核心流程可绕过但体验差）已登记并评估上线风险
- [ ] 本次回归结论：通过 / 不通过

## 8. 建议记录模板

```text
回归版本：
回归日期：
回归人：
阻断问题：
主要问题：
结论：
```
