# Frontend Checklists

## 1. 页面开发检查

- 信息层级清晰（标题、内容、操作、反馈）
- 组件状态完整（默认/悬停/激活/禁用/加载/空态/错误）
- 表单校验可读且可操作
- 危险操作有二次确认

## 2. Vue/Vite 检查

- `npm run build` 通过
- 路由变更后菜单和守卫行为正常
- 未引入循环依赖和明显冗余依赖
- 异步请求有 loading 与错误处理

## 3. 小程序检查

- 页面已在 `app.json` 注册
- 页面跳转参数与回跳逻辑正确
- 授权失效处理正常
- 网络异常和超时文案明确

## 4. 兼容性检查

- 在目标浏览器版本中验证关键流程
- 检查布局、滚动、输入、事件行为
- 必要时提供降级方案而非强行一致

## 5. 性能检查

- 首屏关键资源体积可控
- 图片体积和尺寸匹配场景
- 列表渲染避免不必要重绘
- 避免主线程长任务阻塞交互

## 6. 交付检查

- 说明“改了什么、为什么改”
- 提供可复现实测步骤
- 标注未覆盖风险点
